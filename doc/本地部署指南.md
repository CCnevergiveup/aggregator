# 本地部署指南（完整版 process.py + 榜单脚本 / Windows / uv / 不依赖服务器）

> 适合你现在的情况：**没有服务器，只在本机跑**。目标是在本机自动从公开来源爬取免费节点 → 聚合成一个大节点池 → 用 `rank.py` 筛出两份可直接导入客户端的榜单：
>
> - **干净家宽榜**：`data/clash_clean.yaml`，只保留住宅/家宽 IP，国家分布尽量均衡，取延迟最低的前 100；
> - **速度榜**：`data/clash_fast.yaml`，全部存活节点里，纯按延迟从低到高取前 100。

---

## 〇、先读：本地跑的关键前提

`process.py` 要访问 GitHub / Google / Yandex / Telegram / 公开网页等境外来源，`rank.py` 还要逐个连接节点测速和查询 IP 类型。**所以本地首次运行前，你必须已经有一个能用的“启动代理”。**

没有任何节点的纯小白，需要先通过朋友、临时试用、公共订阅等方式拿到第一个可用代理，挂上系统代理后再用本项目扩大节点池。这个“先有鸡”的问题绕不开。

建议：

1. 先打开 Clash Verge Rev / FlClash / 其它代理客户端；
2. 确认系统代理已开启；
3. 浏览器能访问 GitHub、Google、Telegram 网页后，再跑本项目。

---

## 一、最终流程

```text
第一次准备：安装 uv → 创建 .venv → 安装 requirements.txt

每次更新：
  1. uv run python -u subscribe/process.py -s config.json --overwrite
       ↓
     全源爬取并聚合 → data/raw.yaml
       ↓
  2. uv run python -u subscribe/scripts/rank.py
       ↓
     测延迟 + 查住宅 IP + 国家均衡
       ↓
     data/clash_clean.yaml
     data/clash_fast.yaml
```

> 爬取阶段较慢，筛选阶段也会逐个测节点，首次运行 10～30 分钟都正常。

## 二、准备环境

| 必需项 | 说明 |
|---|---|
| Python 3.9+ | 建议 3.10 / 3.11 / 3.12 |
| uv | 用来创建虚拟环境和安装依赖，替代 `pip install` |
| 启动代理 | 本地访问境外来源必需 |
| GitHub Token | `crawl.github` 必需，用于提高 GitHub 搜索 API 限额 |
| ipapi.is API Key | 可选但强烈建议，用于识别住宅 / 机房 IP |
| 项目代码 | 当前仓库目录，例如 `D:\Work\Github\aggregator` |

核心二进制本仓库已自带：

- `clash/clash-windows-amd.exe`：mihomo 内核，用于测活、测延迟、住宅 IP 检测；
- `subconverter/subconverter-windows-amd.exe`：用于订阅格式转换。

进入项目目录：

```powershell
cd D:\Work\Github\aggregator

```

## 四、创建 `.env`

在项目根目录新建 `.env` 文件：

```env
# GitHub 搜索 API Token，process.py 的 github 爬虫会读取
GH_TOKEN=你的GitHubToken

# 住宅 IP 检测 Key，rank.py 会读取；没有也能跑，但家宽榜准确性会下降
IPAPI_IS_API_KEY=你的ipapi_is_key

# 爬取阶段跳过存活检测，统一交给 rank.py 做；推荐 true，速度更快、少丢节点
SKIP_ALIVE_CHECK=true

# 是否保留 hysteria2 / vless / hysteria 等新协议；推荐 true
ENABLE_SPECIAL_PROTOCOLS=true
```

### 4.1 GitHub Token 怎么拿

1. 打开 <https://github.com/settings/tokens>；
2. 选择 **Generate new token (classic)**；
3. 过期时间选久一点；
4. 如果只本地爬 GitHub 搜索，不需要额外 scope；如果以后还要推 Gist，就勾选 `gist`；
5. 生成后复制到 `.env` 的 `GH_TOKEN`。

### 4.2 ipapi.is Key 怎么拿

1. 打开 <https://ipapi.is/>；
2. 注册账号；
3. 在控制台复制 API Key；
4. 填入 `.env` 的 `IPAPI_IS_API_KEY`。

## 五、创建完整版配置文件 `config.json`

> 之前文档写 `crawl.json` 是因为它只描述“爬取阶段配置”。但 `process.py -s` 接受任意 JSON 文件名，官方 README 常写 `config.json`。为了避免误解，本指南统一使用 **`config.json`**。

在项目根目录新建 `config.json`：

```json
{
    "crawl": {
        "enable": true,
        "exclude": "",
        "threshold": 2,
        "singlelink": true,
        "github": {
            "enable": true,
            "pages": 5,
            "push_to": ["raw"]
        },
        "google": {
            "enable": true,
            "limits": 100,
            "push_to": ["raw"]
        },
        "yandex": {
            "enable": true,
            "within": 3,
            "pages": 5,
            "push_to": ["raw"]
        },
        "telegram": {
            "enable": true,
            "pages": 5,
            "users": {
                "jichang_list": {
                    "push_to": ["raw"]
                }
            }
        },
        "pages": []
    },
    "groups": {
        "raw": {
            "emoji": false,
            "list": true,
            "targets": {
                "clash": "raw-clash"
            }
        }
    },
    "storage": {
        "engine": "local",
        "items": {
            "raw-clash": {
                "fileid": "raw.yaml",
                "folderid": "data"
            }
        }
    }
}
```

### 5.1 为什么这个配置是“完整版”

它使用的是 `process.py`，不是简化版 `collect.py`，并且启用了多种爬取源：

| 源 | 是否建议开启 | 说明 |
|---|---:|---|
| `github` | 强烈建议 | 当前最稳的公开源之一；必须设置 `GH_TOKEN` |
| `google` | 建议 | 可补充公开网页结果；要求本机网络能访问 Google |
| `yandex` | 建议 | 可补充搜索结果；部分网络环境比 Google 更容易通 |
| `telegram` | 可开但不要依赖 | 帖子里多次提到 `jichang_list` 曾失效；能抓到算加分，抓不到不要惊讶 |
| `pages` | 有明确网页时再开 | 适合你自己找到固定分享页后加入 |
| `repositories` | 进阶再用 | 适合盯某个 GitHub 仓库提交记录 |
| `twitter` | 不推荐小白启用 | 可用性受登录、反爬、网络影响大 |

### 5.2 最容易写错的三段名字

这三处必须串起来：

```text
crawl.*.push_to: ["raw"]
        ↓
groups.raw
        ↓
groups.raw.targets.clash = "raw-clash"
        ↓
storage.items.raw-clash
```

任意一个名字对不上，`process.py` 会报配置错误。

## 六、运行全源爬取

在项目根目录执行：

```powershell
uv run python -u subscribe/process.py -s config.json --overwrite
```

参数说明：

| 参数 | 作用 |
|---|---|
| `-s config.json` | 指定完整配置文件 |
| `--overwrite` | 不读取上一次剩余结果，直接重新生成；本地小白推荐加上，结果更清晰 |
| `-n 32` | 可选，降低并发；网络不稳时加这个 |
| `-t 8000` | 可选，增加超时时间 |

示例：网络不稳时用：

```powershell
uv run python -u subscribe/process.py -s config.json --overwrite -n 32 -t 8000
```

成功后应生成：

```text
data/raw.yaml
```

如果没生成，看第十节排查。

## 七、运行榜单筛选

```powershell
uv run python -u subscribe/scripts/rank.py
```

成功后生成：

```text
data/clash_clean.yaml
data/clash_fast.yaml
```

推荐优先使用：

- 日常使用：`data/clash_clean.yaml`；
- 只追求可用和低延迟：`data/clash_fast.yaml`。

常用参数：

```powershell
# 每份榜单只要 50 个
uv run python -u subscribe/scripts/rank.py -t 50

# 放宽延迟上限到 3000ms
uv run python -u subscribe/scripts/rank.py -d 3000

# 网络弱，降低并发
uv run python -u subscribe/scripts/rank.py -n 16

# 指定输入文件
uv run python -u subscribe/scripts/rank.py -i data/raw.yaml
```

## 八、导入客户端

把下面任意一个文件导入支持 mihomo / Clash.Meta 的客户端：

```text
data/clash_clean.yaml
data/clash_fast.yaml
```

推荐客户端：

- Windows / macOS / Linux：Clash Verge Rev、FlClash；
- Android：FlClash、ClashMetaForAndroid；
- iOS：Shadowrocket / Stash 等。

注意：

1. 不推荐老版 Clash for Windows / ClashX，容易不支持 `hysteria2` / `vless`；
2. 榜单文件是**纯节点列表**，没有完整分流规则；如果客户端要求完整配置，请在客户端里套规则，或再走订阅转换。

## 九、定时自动跑（Windows 任务计划程序）

在项目根目录创建 `run-local-rank.bat`：

```bat
@echo off
cd /d D:\Work\Github\aggregator
uv run python -u subscribe/process.py -s config.json --overwrite
uv run python -u subscribe/scripts/rank.py
```

然后：

1. 打开「任务计划程序」；
2. 创建基本任务；
3. 触发器建议每 1～3 天一次；
4. 操作选择启动 `run-local-rank.bat`；
5. 客户端里手动导入新生成的 `data/clash_clean.yaml`，或使用客户端支持的本地订阅更新方式。

## 十、常见问题排查

| 现象 | 原因 / 解决 |
|---|---|
| `uv` 不是命令 | 安装 uv 后重新打开终端；仍不行就检查 PATH |
| `cannot start crawl from github` | `.env` 里没有 `GH_TOKEN`，或 Token 无效 |
| 爬不到节点 / 全是 timeout | 启动代理没挂好；确认浏览器能访问 GitHub / Google / Telegram |
| `data/raw.yaml` 没生成 | 没抓到任何节点；先确认网络，再调大 `pages`，或补充 `pages` 固定网页源 |
| GitHub 源有效，Telegram 没结果 | 正常；帖子里已反馈 `jichang_list` 频道可能失效 |
| 家宽榜为空 | 大多数免费节点是机房 IP，或 `IPAPI_IS_API_KEY` 未配置 / 被限流 |
| 榜单不足 100 个 | 存活节点不足；调大 `pages`，或降低 `rank.py -d` 限制 |
| `unsupported proxy type` | 客户端太旧，换 Clash Verge Rev / FlClash，或设 `ENABLE_SPECIAL_PROTOCOLS=false` 后重跑 |
| `SyntaxError` | Python 版本低于 3.9，升级 Python |
| `mihomo binary not found` | `clash/` 目录缺少对应平台二进制，或仓库文件不完整 |

## 十一、关于爬取源的推荐结论

结合帖子讨论和当前代码，推荐顺序如下：

1. **GitHub 源作为主力**：配置 `GH_TOKEN`，`pages` 先设 5；结果少再加到 8～10。
2. **Google / Yandex 作为补充**：本地有启动代理时建议都开。
3. **Telegram 不要作为唯一来源**：`jichang_list` 在帖子后期被反馈失效或不稳定，开着可以，但不能只靠它。
4. **有固定分享页再加 `pages`**：这是质量上限最高的方式，但需要你自己维护 URL。
5. **不要过度追求数量**：免费节点寿命短，最终看 `rank.py` 筛出来的可用节点质量，不是 `raw.yaml` 里堆了多少。
